Proyecto: Sistema Integral de Gestión Social y Técnica - Ecosistema VIDA UG (Desarrollado por C-MED)Stack Tecnológico: HTML5/TailwindCSS (Frontend), Python (Backend), Supabase (Base de datos y Storage)1. Resumen y ObjetivosEl objetivo de este proyecto es digitalizar y unificar el flujo de captura de datos para el programa de donación de sillas de ruedas del ecosistema VIDA UG. El sistema permitirá a los capturistas registrar de manera ágil los perfiles socioeconómicos de los beneficiarios y las evaluaciones técnicas/posturales necesarias para la correcta asignación del equipo médico.Objetivos principales:Eliminar el uso de papel en los registros de campo.Centralizar la información (socioeconómica y médica) en una base de datos relacional y escalable en Supabase.Almacenar de forma segura la evidencia fotográfica de las posturas de los pacientes.2. Tipos de Usuario (Personas)Capturista (Usuario Final): Personal de campo o administrativo encargado de realizar las entrevistas y tomar las medidas físicas. No requieren autenticación compleja, solo registrar su nombre para mantener la trazabilidad de los expedientes creados.3. Arquitectura del Sistema (Tech Stack)ComponenteTecnologíaPropósitoFrontendHTML, Tailwind CSS, JS (Vanilla)Vistas responsivas, validación de formularios en el cliente y envío de peticiones (Fetch API) al backend.BackendPython (FastAPI o Flask)API RESTful que recibe los datos, valida la lógica de negocio y se comunica con Supabase de forma segura.Base de DatosSupabase (PostgreSQL)Almacenamiento relacional de los estudios, solicitudes y datos de los beneficiarios.AlmacenamientoSupabase StorageAlojamiento de las fotografías (PNG, JPG) capturadas en los expedientes técnicos.4. Requerimientos Funcionales por MóduloMódulo 1: Acceso al Sistema (Login)Pantalla: Login_Andre.htmlComportamiento:El acceso es directo (sin contraseña).Debe capturar obligatoriamente el Nombre del Capturista.El backend (Python) debe generar un token de sesión temporal o un identificador en el localStorage para asociar todos los registros posteriores a este capturista.Módulo 2: Estudio SocioeconómicoPantalla: code (1).htmlComportamiento y Captura de Datos:Datos del Beneficiario: Nombre, fecha de nacimiento, diagnóstico médico, dirección completa (calle, colonia, ciudad) y teléfonos.Datos de Tutores (1 y 2): Edad, nivel de estudios, estado civil, vivienda, fuente de empleo, ingresos mensuales y afiliación a seguridad social (IMSS/INFONAVIT).Cierre: Captura de ingresos adicionales, historial previo de uso de silla de ruedas, sede y fecha del estudio.Acciones: Guardar borrador (local o en DB con estado "incompleto") y Guardar Estudio definitivo.Módulo 3: Solicitud Técnica (Sillas de Ruedas)Pantalla: code.htmlComportamiento y Captura de Datos:Entorno: Selección del entorno de uso (Urbano, Rural, Mixto).Capacidad Postural: Selección de control de tronco, cabeza y observaciones clínicas (escoliosis, etc.).Medidas Técnicas: Altura total, peso y 5 medidas corporales específicas en pulgadas (cabeza a asiento, hombro a asiento, etc.).Carga Multimedia: Input para subir archivos (imágenes de hasta 10MB). El backend en Python deberá procesar este archivo y subirlo a un bucket en Supabase Storage, obteniendo una URL pública o firmada.Gestión de Donación: Entidad solicitante, prioridad (Alta/Media) y justificación.5. Requerimientos No FuncionalesResponsividad: La interfaz debe ser completamente funcional en dispositivos móviles (tablets y teléfonos), ya que los capturistas suelen trabajar en campo.Integridad de Datos: Validar en el backend (Python) que los campos numéricos (pulgadas, peso, edad, ingresos) tengan el formato correcto antes de insertarlos en Supabase.Manejo de Archivos: Limitar la carga de imágenes estrictamente a .jpg y .png en el frontend y backend para evitar inyecciones de archivos maliciosos.6. Propuesta de Modelo de Datos (Supabase Schema)Se sugieren las siguientes tablas principales en PostgreSQL:capturistas (id, nombre, fecha_registro)beneficiarios (id, nombre, fecha_nacimiento, diagnostico, domicilio, ...)tutores (id, beneficiario_id, nombre, ingreso, empleo, ...)estudios_socioeconomicos (id, beneficiario_id, capturista_id, fecha, sede, ...)solicitudes_tecnicas (id, beneficiario_id, capturista_id, entorno, control_tronco, peso, foto_url, ...)
